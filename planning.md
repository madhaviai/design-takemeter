## Community
Langfuse OSS GitHub issues (langfuse/langfuse). Active LLM observability project with varied issue quality — from detailed self-hosting bug reports to feature asks.

## Labels
- bug_report: broken behavior with evidence
- feature_request: new capability or improvement
- support_question: setup/how-to without clear bug

These matter because maintainers triage differently: bugs need repro, features need scope, questions need docs.

## Edge case
Thin “doesn’t work” posts without repro → support_question unless steps/env are included (then bug_report).
